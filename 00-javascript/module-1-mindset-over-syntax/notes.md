
- An algorithm in programming and web development is a precise, step-by-step set of instructions designed to solve a specific problem or perform a task efficiently. 
- Data structures are essential for organizing, managing, and storing data efficiently to optimize memory usage and speed up program execution
- Big-O notation is a theoretical, mathematical, and abstract way to describe the efficiency and scalability of algorithms. It focuses on how the execution time or memory space requirements of an algorithm increase as the input size (usually denoted as '
') grows towards infinity.

When we talk about algorithm efficiency, we use Big-O notation to describe how performance grows as data increases.

Time Complexity = how long it takes.
O(1) → Constant (access one item)
O(log n) → Logarithmic (binary search)
O(n) → Linear (loop through items)
O(n log n) → Log-linear (merge sort)
O(n²) → Quadratic (nested loops)
Space Complexity = how much memory it needs.

O(1) → Same space
O(n) → Grows with input
Big-O doesn’t measure seconds or bytes — it shows growth rate.

It helps us compare which algorithm scales better as data grows.
Faster code isn’t just about speed — it’s about how well it handles scale.
